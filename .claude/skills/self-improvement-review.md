# Self-Improvement Review

Trigger this review after completing any significant task.

## When to Trigger
- After building a feature or component
- After fixing a bug or resolving an error
- After debugging a complex issue
- After refactoring code
- When user provides corrections or feedback

## Review Process

### Step 1: Assess Learning Value
Ask yourself:
1. Did I encounter any unexpected errors or behaviors?
2. Did I discover a non-obvious solution?
3. Would this knowledge help with similar future tasks?
4. Did the user correct me or suggest a better approach?

If ALL answers are "no", skip the review.

### Step 2: Categorize

**Pitfall** → Save to `.claude/knowledge/pitfalls.md`
**Workflow** → Save to `.claude/knowledge/workflows.md`
**Decision** → Save to `.claude/knowledge/decisions.md`

### Step 3: Format Entry

## [YYYY-MM-DD] Short Description

**Context:** What was the task/situation?
**Learning:** What did you discover?
**Application:** When should this knowledge be used?
**Example:** Brief code or command example

### Step 4: Check for Duplicates
Search target file before adding. Update existing if found.

### Step 5: Update CLAUDE.md (if critical)
For HIGH-IMPACT learnings only (pitfalls >30 min, weekly workflows).
